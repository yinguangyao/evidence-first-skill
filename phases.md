# 阶段启动模板

用途：长任务拆成短对话时，每个阶段新开一个对话，粘贴对应模板。阶段之间用三个
文件交接：`plan.md`（方案与被否掉的选项）、`evidence.md`（素材与来源）、
`todo.md`（遗留项）。任何模型都适用——它只需要"照眼前的模板做一步"。

---

## 阶段 1：澄清

```
任务：<原始任务描述>

请先不要动手。输出：
1. 用你自己的话复述目标，以及做什么/不做什么的边界；
2. 未知项清单，分两栏：A) 你自己能查到的（之后自己去查）；B) 只有我能定的
   （现在一次性问完，不要之后挤牙膏）；
3. 你打算基于哪些假设行动，逐条列出。
```

## 阶段 2：侦察

```
读取 plan.md（如有）。目标：<目标>

去把现场真实过一遍（读原始材料，不要用你的记忆代替）。输出到 evidence.md：
1. 素材清单：每条素材一行，注明来源（哪个文件/哪份原文/哪次运行）；
2. 发现的矛盾：来源之间互相冲突的地方（记住来源连坐规则）；
3. 缺口清单：还缺什么证据、打算去哪补、补不到怎么标注；
4. 同类问题的既有范例：找到的话记录它的骨架。
```

## 阶段 3：定方案

```
读取 evidence.md。目标：<目标>

输出到 plan.md：
1. 2~3 个候选方案，各自的代价和风险；
2. 你的明确推荐及理由（不要把选择题抛回给我）；
3. 被否掉的方案和否掉的理由（留痕）。
方案里每个关键判断标注依据来自 evidence.md 的哪一条。
```

## 阶段 4：执行（每个小步一轮）

```
读取 plan.md 和 todo.md。本步目标：<小步目标>

只做这一步。完成后输出：
1. 做了什么（具体到文件/章节/数据）；
2. 本步的验证方式和验证结果（实际做过的，不是打算做的）；
3. 过程中发现的、可能推翻方案的新信息（有就停下来标注，等我决定）；
4. 更新 todo.md。
```

## 阶段 5：交付

```
读取 plan.md、evidence.md、todo.md。

产出最终交付，要求：
1. 第一句话就是结论；
2. 每个论断和数字标注来源；
3. 附验证方式清单：每项声称"完成/成立"的内容，写明是怎么验证的；
4. 显式列出遗留项、待确认项、已知局限；
5. 写给"没有旁观过程、刚回来的人"，不用回看任何中间对话就能懂。

最后逐条回答 methodology.md 第六节自检清单，贴出检查结果。
```
